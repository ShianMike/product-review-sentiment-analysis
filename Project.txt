I. INTRODUCTION

1.1. Background of the Study
Customer product reviews are a major source of information that influences purchase decisions and reflects customer satisfaction. Modern e-commerce and retail environments generate a large volume of reviews, making it difficult for businesses to manually read, summarize, and identify recurring concerns quickly. Consumer research has shown that reviews are widely used and considered essential in purchase decisions, highlighting the importance of extracting insights from review content efficiently (PowerReviews, 2023).

Data analytics is suitable for this problem because product reviews are unstructured text that can be transformed into structured insights through natural language processing (NLP) and machine learning. Sentiment analysis helps classify reviews into sentiment categories (positive, neutral, negative), while text mining can extract themes such as common complaints and praises. This enables businesses to track customer satisfaction trends over time and prioritize improvement areas based on evidence rather than manual guessing (Tan et al., 2023; Mao et al., 2024).

Despite the availability of reviews, current challenges include the time required for manual analysis, inconsistent interpretation across people, and delayed detection of recurring product issues. These inefficiencies can negatively affect decision-making related to product improvement, customer support response, and quality assurance.
1.2. Problem Statement
Businesses and sellers face difficulty in efficiently interpreting large volumes of product review text to identify overall customer sentiment and recurring product issues. Reviews often contain valuable feedback, but manual analysis is time-consuming and may cause delays in detecting common complaints that require immediate action. Since reviews strongly influence consumer decisions, the inability to quickly extract actionable insights can reduce customer satisfaction and competitiveness (PowerReviews, 2023).
This project addresses the need for a web-based system that can automatically analyze product review datasets, classify sentiment, summarize key themes, and present insights through an interactive dashboard for decision support.

IV. REVIEW OF RELATED LITERATURE AND SYSTEMS

4.1. Related Studies

Sentiment analysis is a well-established area in NLP that focuses on extracting and classifying opinions expressed in text into polarity categories such as positive and negative. Liu (2012) explains that sentiment analysis (opinion mining) aims to identify and analyze opinions, sentiments, and evaluations from textual data, which makes it suitable for analyzing product reviews.

Recent literature shows that sentiment analysis pipelines typically include preprocessing, feature representation (e.g., TF-IDF or embeddings), and classification using machine learning, deep learning, or ensemble learning approaches (Tan et al., 2023; Mao et al., 2024). For product reviews, transformer-based and hybrid approaches are widely studied due to their ability to capture context and improve classification performance compared with simpler baselines (Ghatora et al., 2024; Daza et al., 2024).

Recent work in e-commerce review analytics highlights the value of extracting not only overall sentiment but also fine-grained, aspect-based sentiment to better understand drivers of customer satisfaction (Davoodi et al., 2025; Wankhade et al., 2024).

4.2. Related Systems

Existing tools provide review analytics, but many require paid subscriptions or platform-specific integration. General analytics dashboards often focus on numerical KPIs and may not directly provide sentiment insights from unstructured text. This proposed system focuses on a lightweight, upload-based workflow for sentiment analysis and thematic summarization, suitable for small businesses or academic prototyping, while emphasizing interpretability and evaluability.

V. CONCEPTUAL FRAMEWORK
Input
The system accepts a customer review dataset (CSV/Excel) containing review text and optional fields such as rating, date, and product/category.
Process
The system validates the dataset, preprocesses the text, performs overall sentiment classification and aspect-based sentiment analysis (e.g., delivery, quality, price), extracts themes/keywords, and aggregates results by product and time.
Output
The system generates a web dashboard that displays overall sentiment distribution, aspect-level sentiment breakdown, sentiment trends (if dates exist), top complaint and praise themes, and provides filtering and export features.

VI. METHODOLOGY

6.1. Data Source and Description
•	Dataset Source: Public datasets from repositories such as Kaggle.
•	Type of Data: Primarily unstructured text (reviews) with optional structured columns (rating, date, product/category).
•	Attributes/Features: Review text, predicted sentiment label, sentiment score, extracted keywords/themes, and optional metadata.

6.2. Data Preprocessing

•	Text cleaning (lowercasing, punctuation removal, whitespace normalization).
•	Tokenization and stopword removal.
•	Handling missing values by removing empty reviews or marking missing metadata fields.
•	Transforming text into numerical features (e.g., TF-IDF vectors) for classification.

6.3. Data Analytics Techniques

•	Chosen Technique: Sentiment Classification and Aspect-Based Sentiment Analysis (ABSA)
•	Justification: Overall sentiment supports quick satisfaction monitoring, while ABSA identifies which product aspects drive negative/positive sentiment for better prioritization. (Tan et al., 2023; Mao et al., 2024).
•	Supporting Technique: Text mining for aspect/theme extraction and keyword/phrase summarization.

6.4. System Development Methodology

The project will use an Agile/Iterative approach with incremental development:
1.	Requirements and dataset preparation
2.	Prototype NLP pipeline and baseline classifier
3.	Web dashboard development
4.	Integration and testing
5.	Evaluation and documentation

VII. SYSTEM ARCHITECTURE AND DESIGN

7.1. System Architecture

•	Frontend: HTML/CSS/JavaScript dashboard UI for upload, filters, and visualization.
•	Analytics Engine: Python and/or R scripts for preprocessing, sentiment classification, aspect-based sentiment analysis, theme extraction, and evaluation.
•	Storage: No database; file-based inputs and outputs (e.g., processed CSV exports and summary JSON for charts).

7.2. Functional Requirements

•	The system shall allow users to upload CSV/Excel review datasets.
•	The system shall validate required columns (at minimum: review text).
•	The system shall preprocess text and perform sentiment classification and aspect-based sentiment analysis.
•	The system shall compute overall and aspect-level sentiment summaries and extract key themes/aspects.
•	The system shall present results in a dashboard with filters and export features.

7.3. Non-Functional Requirements

•	Performance: The system should process a moderate dataset within reasonable time for classroom-scale use.
•	Usability: The interface should be simple with clear labels, charts, and filters for non-technical users.
•	Security (basic): Uploaded datasets should not be publicly exposed; the system should avoid storing sensitive identifiers and should support local/offline use if required.
